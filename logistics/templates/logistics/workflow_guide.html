LCL and FCL Procedures

Download (HTML)
Operational blueprint for Groupage (LCL) and Full Container (FCL)

This UI guide reflects the full procedures for receiving, verification, shipment planning, loading, transit, arrival, delivery, and billing. It also standardizes shared status, movement logging, inventory transitions, and finance integration rules.

Status Engine: RECEIVED VERIFIED ALLOCATED / BOOKED LOADED IN TRANSIT ARRIVED DELIVERED / COMPLETED
1. Groupage (LCL) Procedures
A.Cargo Receiving (Inbound to Warehouse)
  • Capture Entry Number, Client, Cargo Description, Packages, Weight, CBM, and Condition (Damaged or OK).
  • Assign temporary warehouse location (rack or shelf).
  • Set status to RECEIVED.
B.Cargo Verification and Measurement
  • Re-weigh and re-measure CBM to avoid disputes.
  • Confirm chargeable unit as max(weight, volume).
  • Set status to VERIFIED.
C.Cargo Consolidation Planning
  • Create or select a Groupage Shipment.
  • Define destination, cut-off date, and container capacity (CBM and weight limits).
D.Cargo Allocation to Shipment
  • Assign multiple cargo entries to one shipment.
  • Auto-sum total weight and CBM.
  • Prevent overloading using capacity validation.
  • Set status to ALLOCATED.
E.Packing and Container Loading
  • Record loading date, container number, and loader.
  • Update all cargo items to LOADED.
F.Manifest Generation
  • Auto-generate shipment manifest for all cargo entries.
  • Include Entry Number, Client, Description, and Weight or CBM totals.
G.Dispatch and Transit
  • Update shipment departure date and driver or transport details.
  • Set Shipment status to IN TRANSIT.
  • Set Cargo status to IN TRANSIT.
H.Arrival and Deconsolidation
  • At destination warehouse, mark shipment as ARRIVED.
  • Break down and separate cargo per client.
  • Set cargo status to READY FOR DELIVERY.
I.Delivery to Client
  • Capture receiver name, date, and proof of delivery.
  • Set status to DELIVERED.
J.Billing (Critical)
  • Per cargo item: chargeable weight or CBM, rate, and storage if delayed.
  • Generate invoice per client.
2. Full Container Load (FCL) Procedures

FCL is simpler because one client uses one container.

A.Booking and Order Creation
  • Capture client, container type (20ft or 40ft), cargo type, origin, and destination.
  • Set status to BOOKED.
B.Container Assignment
  • Assign container number and seal number.
  • Link container to client shipment.
C.Cargo Loading (Direct or Warehouse)
  • Support direct stuffing (client premises) and warehouse stuffing.
  • Capture loading date, supervisor, and cargo summary.
  • Set status to LOADED.
D.Documentation
  • Commercial invoice
  • Packing list
  • Bill of lading
E.Dispatch and Shipping
  • Record vessel or truck and departure date.
  • Set status to IN TRANSIT.
F.Tracking
  • Update milestones: port arrival, customs clearance, inland movement.
G.Delivery and Container Release
  • At destination, clear container and deliver to client or yard.
  • Capture return of empty container.
  • Set status to COMPLETED.
H.Billing
  • Freight
  • Container handling
  • Port charges
  • Demurrage when delayed
3. Shared Components (LCL and FCL)
A. Status Engine (Important)

Standardized statuses across the system:

RECEIVED VERIFIED ALLOCATED / BOOKED LOADED IN TRANSIT ARRIVED DELIVERED / COMPLETED
B. Movement Log (Audit Trail)
  • Every action records Cargo ID or Shipment ID.
  • From location to location transition.
  • Timestamp and user.
C. Inventory Logic
  • Before shipment: Warehouse stock.
  • After allocation: Reserved stock.
  • After loading: In transit stock.
  • After delivery: Out of system.
D. Financial Integration
  • Link cargo to charges to invoice.
  • LCL billing is per cargo item.
  • FCL billing is per shipment.
4. Key Design Difference (LCL vs FCL)
Feature LCL (Groupage) FCL (Full Container)
Ownership Multiple clients Single client
Billing Per cargo item Per container
Complexity High Moderate
Tracking Item-level plus shipment-level Shipment-level
Warehouse Need Required Optional
